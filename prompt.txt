# Car Rental Website Project - Complete Context

## Project Overview
I'm building a full-stack car rental website called "Sihabi Cars" for a Moroccan car rental company. The tech stack is:
- **Frontend**: React + TypeScript + Vite + Tailwind CSS + shadcn/ui + Framer Motion
- **Backend**: Node.js + Express + SQLite
- **Email**: Nodemailer with Gmail SMTP
- **Multi-language**: French (default), Arabic, English

## Database Schema (SQLite)

### users table
- id, email, password (bcrypt hashed)
- first_name, last_name, phone, city
- cin, driver_license (newly added)
- role (user/admin), created_at
- reset_token, reset_token_expiry (for password reset)
- manual_spent_adjustment (for admin adjustments)

### cars table
- id, name, brand, category
- price_per_day, image_url, description
- seats, transmission, fuel_type
- is_available, is_featured
- created_at, updated_at

### bookings table
- id, car_id, user_id
- customer_name, customer_email, customer_phone
- cin, driver_license (customer documents)
- start_date, end_date, total_price
- payment_method (cash/card), status (pending/confirmed/completed/cancelled)
- language (fr/ar/en - for emails)
- created_at

### contact_messages table
- id, name, email, phone, message
- status (unread/read/replied)
- created_at

## Key Features Implemented

### 1. Authentication System
- Login/Register with JWT tokens
- Password reset via email with secure tokens (1-hour expiry)
- Dedicated login page at `/login`
- Auth modal for quick access
- Protected routes for admin and user areas

### 2. Multi-Language Support (FR/AR/EN)
- Language switcher in navbar
- All UI elements translated
- Emails sent in user's selected language
- RTL support for Arabic

### 3. Car Management
- Browse cars with filters and search
- Car details page with booking form
- Admin can add/edit/delete cars
- Featured cars on homepage
- Availability status

### 4. Booking System
- Date range picker with validation
- Auto-fill user info (name, email, phone, CIN, driver license) from profile
- Payment method selection (cash/card - both paid at pickup)
- CIN and driver license fields required
- Redirect to "My Bookings" after successful booking
- Users can cancel pending bookings

### 5. Email Notification System (Nodemailer + Gmail SMTP)
**4 Email Types:**
- **Booking Confirmation** - Sent to customer on new booking
- **Admin Alert** - Sent to admin on new booking
- **Status Change** - Sent when admin changes booking status
- **24h Reminder** - Automated cron job sends reminders before pickup

API Endpoints
Auth Routes ( /api/auth)
POST /login - User login

POST /register - User registration

POST /forgot-password - Request password reset

GET /validate-reset-token - Validate reset token

POST /reset-password - Reset password with token

Car Routes ( /api/cars)
GET / - Get all cars (with filters)

GET /:id - Get car by ID

POST / - Create car (admin only)

PUT /:id - Update car (admin only)

DELETE /:id - Delete car (admin only)

Booking Routes ( /api/bookings)
GET / - Get all bookings (admin only)

GET /my - Get user's bookings

GET /:id - Get booking by ID

POST / - Create booking

PATCH /:id/status - Update booking status (admin only)

DELETE /:id - Cancel booking

User Routes ( /api/users)
GET /me - Get current user profile

PUT /me - Update current user profile

GET / - Get all users (admin only)

GET /analytics - Get analytics data (admin only)

PUT /:id - Update user (admin only)

Contact Routes ( /api/contact)
POST / - Submit contact form
